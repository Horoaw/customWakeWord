# PIPELINE — customWakeWord Technical Execution

The exact technical recipe. Every command, every config, every file path. Designed so Claude Code executes autonomously and a future maintainer can reproduce.

This doc walks the **Tofu worked example** end-to-end. For *any* wake word, substitute the slug — every command below works with `--project <slug>` interchangeably. See [`README.md`](README.md) for the generic-toolkit framing and [`EXAMPLES.md`](EXAMPLES.md) for other worked examples.

Training delegates to upstream [`OHF-Voice/micro-wake-word`](https://github.com/OHF-Voice/micro-wake-word) (formerly `kahrendt/microWakeWord`). Our scripts are thin wrappers that produce the data + configs upstream expects, then call its training/export entry points.

## Stage 0 — Local Mac Bootstrap

```bash
brew install python@3.10 git git-lfs ffmpeg sox

cd ~/Documents/Github/tofuWakeWord
python3.10 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

Python **must be 3.10** (microwakeword pins it; upstream issue #62 documents the 3.11 break). On RunPod we pin to `runpod/pytorch:2.4.0-py3.10-cuda12.4.1-devel-ubuntu22.04`.

Mac runs: TTS synthesis (CPU/MPS), eval against an existing `.tflite`. Mac does NOT run: training. Use RunPod.

## Stage 1 — Data Pipeline

### 1a. Positives via Piper

Script: `scripts/synth_positives.py`
- Reads `configs/examples/<slug>/wake_phrases.yaml` (generated by `scripts/init_wake.py`).
- English projects auto-clone `piper-sample-generator` v3 and use `en_US-libritts_r-medium.pt`.
- Multilingual projects use standard Piper ONNX voices directly through `piper-tts`; Mandarin projects receive two default voices and do not download the English generator.
- For each phrase, generates the configured number of samples and length-scale variants into `data/<slug>/synth/positives/<phrase_slug>/`.
- Output: one WAV per row + unified `manifest.jsonl`.

```bash
python scripts/synth_positives.py --project tofu
```

Tofu default targets (from `configs/examples/tofu/wake_phrases.yaml`):
- "hey tofu" — 5,000
- "hi tofu" — 2,500
- "hello tofu" — 1,500
- "okay tofu" — 1,000

### 1b. Hard negatives

Script: `scripts/synth_hard_negatives.py`
- Reads `configs/examples/<slug>/hard_negatives.yaml`.
- Same engine, iterates over each bucket × phrase.
- Output tagged with `bucket_id` so per-bucket FAR is measurable.

```bash
python scripts/synth_hard_negatives.py --project tofu
```

Optional LLM-driven extension (recommended — better than hand-rolled rhyme heuristics):
```bash
source scripts/load_creds.sh   # needs TOGETHER_API_KEY
python scripts/suggest_hard_negatives.py --name tofu
```

### 1c. Bulk negatives — upstream HF mmap dataset

Script: `scripts/download_hf_negatives.py`
- Pulls 7 zips from `huggingface.co/datasets/kahrendt/microwakeword` (~9 GB total): `speech`, `speech_background`, `dinner_party`, `dinner_party_background`, `dinner_party_eval`, `no_speech`, `no_speech_background`.
- Already pre-computed spectrograms in RaggedMmap format — no feature extraction needed.

```bash
python scripts/download_hf_negatives.py --out data/negative_datasets
```

License note: CC-BY-NC-4.0 (non-commercial). For commercial deployments, swap to MUSAN + CV + AudioSet via `scripts/collect_negatives.py` (kept as legacy path).

### 1d. Feature extraction (positives + hard-negs → RaggedMmap)

Script: `scripts/build_features.py`
- For each positive/hard-neg WAV:
  1. Apply `microwakeword.audio.augmentation.Augmentation` (canonical recipe: SevenBandParametricEQ, TanhDistortion, PitchShift, BandStopFilter, AddColorNoise, AddBackgroundNoise@0.75 p, Gain@1.0 p, RIR@0.5 p; SNR -5 to +10 dB).
  2. Generate spectrograms via `microwakeword.audio.spectrograms.SpectrogramGeneration` (slide_frames=10 for train, =1 for val/test; step_ms=10).
  3. Write to `mmap_ninja.ragged.RaggedMmap`.
- Output: `data/<slug>/features/{training,validation,testing}/wakeword_mmap/` + `data/<slug>/hard_negatives_features/...`.

```bash
python scripts/build_features.py --project tofu --download-aug-corpora
```

`--download-aug-corpora` fetches MIT IR survey + FMA-XS music + AudioSet shard (the corpora the canonical notebook uses). One-time per environment, ~5 GB.

## Stage 2 — Training (upstream)

### 2a. Training config

Per-project YAML at `configs/examples/<slug>/training_parameters.yaml`. Schema matches what `microwakeword.model_train_eval.load_config()` expects. The Tofu config (`configs/examples/tofu/training_parameters.yaml`) uses:
- 6 feature corpora: tofu positives, tofu hard-negs (`penalty_weight: 5.0`), speech, dinner_party (`sampling_weight: 15.0, penalty_weight: 3.0`), no_speech, dinner_party_eval (val/test only).
- `training_steps: [10000]`, `batch_size: 128`, `learning_rates: [0.001]`.
- `negative_class_weight: [20]` (positives are minority class).
- `clip_duration_ms: 1500`, `window_step_ms: 10`.
- `maximization_metric: average_viable_recall` with `target_minimization: 0.5` FA/hr.

### 2b. Run training

Script: `scripts/train_microwakeword.py` — thin wrapper around upstream.

Builds the argparse Namespace for `microwakeword.model_train_eval` (subcommand `mixednet`, architecture flags `pointwise_filters="64,64,64,64"`, `mixconv_kernel_sizes="[5],[7,11],[9,15],[23]"`, `stride=3`), calls `load_config(flags, mixednet)`, then `train_model(...)` then `evaluate_model(..., test_tflite_streaming_quantized=True)`.

```bash
python scripts/train_microwakeword.py --project tofu \
    --training-config configs/examples/tofu/training_parameters.yaml
```

Output:
- `trained_models/tofu/best_weights.weights.h5`
- `trained_models/tofu/tflite_stream_state_internal_quant/stream_state_internal_quant.tflite` (the deployable artefact)
- `models/tofu-wakeword-v0.tflite` (copied to our canonical location)
- Operating-point table printed to stdout (cutoff vs recall vs FA/hr)

### 2c. RunPod launcher

Script: `scripts/runpod_train.py` — orchestrates the whole pipeline on a pod.

```bash
source scripts/load_creds.sh
python scripts/runpod_train.py --project tofu --hf-repo-id <user>/tofu-wakeword-v0
```

The launcher:
1. Provisions RTX 4090 SECURE @ ~$0.69/hr (cap MAX_WAIT_S=4h → max ~$2.76).
2. Pins `runpod/pytorch:2.4.0-py3.10` image (Python 3.10 to dodge issue #62).
3. Starts log server on `:8001` serving `/workspace/` — `setup.log` survives any subshell crash.
4. Drives the full pipeline on-pod: install → synth (positives + hard-negs) → download HF negatives → build features → train → emit manifest → optional HF upload.
5. Runs held-out eval, manifest generation, and upload only when eval tasks are available. Otherwise it exports the model and intentionally skips release metadata.
6. Auto-stops on `/workspace/_done` marker.

## Stage 3 — Eval + threshold pick

The trainer prints a cutoff table. Pick the highest threshold where `recall ≥ 0.95` AND `far/hr ≤ 0.5`. Then run our held-out eval:

```bash
python -m eval.runner --project tofu \
    --model models/tofu-wakeword-v0.tflite \
    --threshold <picked-cutoff> \
    --out eval/results/tofu-v0__$(date +%s).json
```

Metrics: FRR (positives), FAR/hr (bulk), per-bucket FAR (hard-negs). Reads tasks from `eval/tasks/tofu/` (seed via `scripts/seed_eval_tasks.py`).

## Stage 4 — Manifest + release

### 4a. ESPHome v2 manifest JSON

Script: `scripts/emit_manifest.py`

```bash
python scripts/emit_manifest.py --project tofu --threshold 0.85
```

Writes `configs/examples/tofu/manifest.json` in the v2 schema:
```json
{
  "type": "micro",
  "wake_word": "Tofu",
  "model": "tofu-wakeword-v0.tflite",
  "trained_languages": ["en"],
  "version": 2,
  "micro": {
    "probability_cutoff": 0.85,
    "feature_step_size": 10,
    "sliding_window_size": 5,
    "tensor_arena_size": 30000,
    "minimum_esphome_version": "2024.7.0"
  }
}
```

### 4b. HuggingFace upload

Script: `scripts/upload_to_hf.py`

```bash
python scripts/upload_to_hf.py --project tofu \
    --model models/tofu-wakeword-v0.tflite \
    --repo-id <user>/tofu-wakeword-v0 \
    --eval-json eval/results/tofu-v0__<latest>.json \
    --esphome configs/examples/tofu/manifest.json
```

### 4c. GitHub release

```bash
gh release create v0.1.0 \
    models/tofu-wakeword-v0.tflite \
    configs/examples/tofu/manifest.json \
    eval/results/tofu-v0__<latest>.json \
    --title "tofuWakeWord v0.1.0"
```

## Stage 5 — ESPHome deploy

Drop the manifest into any ESPHome config for an ESP32-S3 device:

```yaml
micro_wake_word:
  microphone: tofu_mic
  vad:
  models:
    - model: tofu
      probability_cutoff: 0.85
      sliding_window_size: 5
      url: https://huggingface.co/<user>/tofu-wakeword-v0/resolve/main/tofu-wakeword-v0.tflite
```

Then `esphome run tofu.yaml`. Full ESP32-S3 + INMP441 example in `configs/templates/esphome_template.yaml`.

## Tool Reference

| Stage | Tool | Why |
|---|---|---|
| TTS | Piper ONNX (`piper-tts`) or piper-sample-generator (English LibriTTS-R) | Lightweight multilingual synthesis plus the canonical English recipe |
| Bulk audio | HF mmap zips (kahrendt/microwakeword) | Pre-computed, immediately usable |
| Augmentation | `microwakeword.audio.augmentation` | Same code path the on-device runtime expects |
| Features | `microwakeword.audio.audio_utils.generate_features_for_clip` (C `audio_microfrontend` via pymicro-features) | Bit-exact match to on-device frontend |
| Architecture | MixedNet `64,64,64,64` | Ships as Okay Nabu, Hey Jarvis in ESPHome v2 |
| Training | upstream `microwakeword.model_train_eval` | Only path that produces a deployable streaming `.tflite` |
| Quantization | upstream `convert_saved_model_to_tflite(..., quantize=True)` with `_experimental_variable_quantization=True` | Streaming-state variables quantized → no Quantize/Dequantize ops on device |
| Runtime | TFLite Micro via ESPHome `micro_wake_word` | ESP32-S3 native |
| Provisioning | RunPod 4090 SECURE | $0.69/hr, fastest converging point for this scale |
| Storage | HuggingFace Hub | Free, public |
